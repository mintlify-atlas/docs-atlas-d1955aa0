# Book Store API Automation Testing

A complete API test automation framework built with **Rest Assured**, **Cucumber BDD**, and **JUnit**. This framework validates Place Management REST API endpoints — Add, Get, and Delete — using a structured BDD approach with reusable components and detailed logging.

## What this framework does

This framework automates the testing of a **Place Management REST API** that supports three core operations:

| Operation | HTTP Method | Endpoint |
|-----------|-------------|----------|
| Add Place | POST | `/maps/api/place/add/json` |
| Get Place | GET | `/maps/api/place/get/json` |
| Delete Place | POST | `/maps/api/place/delete/json` |

Tests are written in **plain English using Gherkin syntax** (BDD), making them readable by non-technical stakeholders. The framework validates end-to-end scenarios such as creating a place, verifying it exists via its returned `place_id`, and deleting it.

## Framework architecture at a glance

```
Feature File (Gherkin)
        ↓
  TestRunner.java          ← Runs tests via @RunWith(Cucumber.class)
        ↓
StepDefinition.java        ← Maps Gherkin steps to Java methods
        ↓
   Utils.java              ← Shared RequestSpecification (base URL, headers, logging)
        ↓
TestDataBuild.java         ← Builds request body using POJO classes
        ↓
APIResources.java          ← Enum provides endpoint paths
        ↓
  Rest Assured             ← Sends HTTP request & validates response
        ↓
 logging.txt               ← Request & response logged automatically
        ↓
cucumber-report.json       ← JSON report → HTML report via Maven plugin
```
