## Prerequisites

Before you begin, make sure you have the following installed and available:

- **Java 8 or higher** — the test suite requires a JDK, not just a JRE
- **Apache Maven 3.6+** — used to manage dependencies and run tests
- **Eclipse IDE** (with the M2E Maven plugin) or **IntelliJ IDEA**
- **Internet access** to `rahulshettyacademy.com` — all API calls target this host during test execution

## Setup

<Steps>

<Step title="Clone the repository">

Clone the project to your local machine:

```bash
git clone https://github.com/your-org/book-store-api-automation.git
cd book-store-api-automation
```

</Step>

<Step title="Import into Eclipse">

1. Open Eclipse and go to **File → Import**.
2. Select **Maven → Existing Maven Projects** and click **Next**.
3. Click **Browse** and select the cloned project directory.
4. Confirm that `pom.xml` appears in the projects list, then click **Finish**.

Eclipse resolves Maven dependencies automatically. Wait for the import and build to complete before proceeding.

<Tip>
If you use IntelliJ IDEA, open the project directory directly. IntelliJ detects `pom.xml` and imports it as a Maven project automatically.
</Tip>

</Step>

<Step title="Fix the hardcoded file path in Utils.java">

The `getGlobalValue` method in `src/test/java/resources/Utils.java` contains an absolute path pointing to the original developer's machine. You must update it to match your system.

<Warning>
`Utils.java` contains a hardcoded Windows absolute path:

```java
FileInputStream file = new FileInputStream(
    "C:\\Users\\a\\eclipse-workspace\\APIFramework\\src\\test\\java\\resources\\global.properties"
);
```

Running the tests without fixing this line throws a `FileNotFoundException` at runtime.
</Warning>

Replace that line with a path relative to the project root. Because `global.properties` is located at `src/test/java/resources/global.properties`, update the method to:

```java
public static String getGlobalValue(String key) throws IOException {
    Properties prop = new Properties();
    FileInputStream file = new FileInputStream(
        "src/test/java/resources/global.properties"
    );
    prop.load(file);
    return prop.getProperty(key);
}
```

Maven resolves relative paths from the directory containing `pom.xml`, which is the project root. Always run `mvn` commands from that directory.

</Step>

<Step title="Install dependencies">

Download all Maven dependencies declared in `pom.xml` without running tests:

```bash
mvn clean install -DskipTests
```

This pulls in Rest Assured, Cucumber, JUnit, Jackson Databind, Groovy, and the `maven-cucumber-reporting` plugin. A successful run ends with `BUILD SUCCESS`.

</Step>

<Step title="Run all tests">

Execute the full test suite:

```bash
mvn test
```

Maven discovers `TestRunner.java`, which points Cucumber at the feature files under `src/test/java/features` and step definitions under the `stepDefinitions` package. Both scenarios in `placeValidations.feature` run — one tagged `@AddPlace @Regression` and one tagged `@DeletePlace @Regression`.

A raw JSON report is written to `target/jsonReports/cucumber-report.json` after the run.

<Note>
The API key `qaclick123` is hardcoded as a query parameter in `Utils.java`:

```java
.addQueryParam("key", "qaclick123")
```

All requests to the Place Management API are sent with this key automatically. No additional configuration is required unless the key changes.
</Note>

</Step>

<Step title="Generate the HTML report">

Run the `maven-cucumber-reporting` plugin to produce a human-readable HTML report from the JSON output:

<CodeGroup>

```bash Run tests and generate report
mvn verify
```

```bash Generate report without re-running tests
mvn verify -DskipTests
```

</CodeGroup>

</Step>

<Step title="View the report">

Open the generated report in your browser:

```
target/cucumber-html-reports/overview-features.html
```

The overview page shows a feature-by-feature summary with pass/fail counts, scenario tags (`@AddPlace`, `@DeletePlace`, `@Regression`), and step-level details for each scenario.

</Step>

</Steps>
